# 스트라우만 세미나 통합 관리 시스템 - 제품 요구사항 문서

## 1. 프로젝트 개요

### 1.1 프로젝트 명
- **스트라우만 세미나 통합 관리 시스템**

### 1.2 프로젝트 배경
- 기존 PHP 기반의 스트라우만 캠퍼스 시스템에 세미나 관리 기능을 추가
- 회원 전용 세미나 신청, 결제, 출석 관리, 설문조사, 수료증 발급 기능을 통합 제공
- 현장 스태프를 위한 QR 스캔 출석 체크 기능 제공

### 1.3 프로젝트 목표
- 회원에게 일관된 UX를 제공하는 세미나 관리 시스템 구축
- 관리자의 효율적인 세미나 운영 및 참여자 관리 지원
- 현장 스태프를 위한 모바일 친화적 출석 관리 기능 제공

## 2. 대상 사용자

### 2.1 회원 (일반 사용자)
- 세미나 목록 조회
- 세미나 신청 및 결제
- 마이페이지에서 신청 내역 확인
- QR코드를 통한 출석 체크
- 수료증 다운로드
- 설문조사 참여

### 2.2 관리자
- 세미나 생성 및 관리
- 신청자 관리
- 출석 관리
- 수료증 발급 관리

### 2.3 현장 스태프 (Staff)
- QR 스캔을 통한 출석 체크
- 참여자 명단 확인

## 3. 핵심 기능 요구사항

### 3.1 세미나 목록 및 상세 조회
**기능 개요**
- 진행 중인 세미나 목록을 카드 형태로 표시
- 세미나 상세 정보 확인

**상세 요구사항**
1. **목록 페이지**
   - 썸네일/포스터 이미지 표시
   - 세미나 제목, 일시, 장소, 정원, 가격 표시
   - 신청 마감 임박 표시
   - 페이지네이션 또는 무한 스크롤

2. **상세 페이지**
   - 세미나 상세 정보 (제목, 설명, 일시, 장소, 정원, 가격)
   - 신청 기간 표시
   - 현재 신청 인원 / 정원 표시
   - 신청 버튼 (신청 기간 내 & 정원 미달 시 활성화)

### 3.2 세미나 신청 및 결제
**기능 개요**
- 회원이 세미나를 신청하고 결제를 완료

**상세 요구사항**
1. **신청 전**
   - 로그인 여부 확인 (비회원은 로그인 페이지로 이동)
   - 중복 신청 방지 체크
   - 정원 확인

2. **신청 프로세스**
   - 개인정보 확인 및 수정
   - 결제 방법 선택 (신용카드, 가상계좌 등)
   - 결제 진행
   - 신청 완료 후 확인 페이지 표시

### 3.3 마이페이지 (신청 내역)
**기능 개요**
- 내 신청 내역 조회 및 관리

**상세 요구사항**
1. **신청 내역 목록**
   - 신청일, 세미나명, 일시, 상태 (결제완료/결제대기/취소/환불) 표시
   - 상태별 필터링

2. **신청 상세**
   - 세미나 정보
   - 신청 정보 (일시, 상태)
   - QR코드 (출석 체크용)
   - 수료증 다운로드 버튼 (출석 완료 시 활성화)
   - 취소/환불 요청 버튼 (특정 조건 하에 활성화)

### 3.4 QR코드 출석 체크
**기능 개요**
- 참여자의 QR코드를 스캔하여 출석 체크

**상세 요구사항**
1. **회원용 QR코드**
   - 마이페이지에서 세미나별 QR코드 제공
   - QR코드에는 신청 ID 정보 포함
   - 출석 전까지 유효

2. **스태프용 스캐너**
   - 카메라 접근 권한 요청
   - QR코드 스캔
   - 출석 체크 처리
   - 중복 출석 방지
   - 오프라인 모드 지원 (네트워크 불안정 시)

### 3.5 수료증 발급
**기능 개요**
- 세미나 출석 완료자에게 수료증 제공

**상세 요구사항**
1. **수료증 생성**
   - 세미나 템플릿에 참여자 정보 동적 삽입
   - 세미나명, 일자, 참여자명, 수료 번호 포함
   - PDF 다운로드

2. **발급 조건**
   - 세미나 출석 완료 시 자동 발급
   - 마이페이지에서 다운로드 가능

### 3.6 설문조사
**기능 개요**
- 세미나 만족도 조사

**상세 요구사항**
1. **설문조사 항목**
   - 세미나 내용 만족도
   - 강사 진행 만족도
   - 시설 및 환경 만족도
   - 전반적인 만족도
   - 건의사항 (필수 아님)

2. **진행 방식**
   - 출석 완료 후 설문조사 참여 요청
   - 마이페이지에서 참여 가능
   - 참여 완료 표시

## 4. UI/UX 가이드라인

### 4.1 디자인 원칙
- 모던하고 깔끔한 인터페이스
- 모바일 우선 (Mobile First) 반응형 디자인
- 기존 스트라우만 캠퍼스 디자인 가이드라인 준수

### 4.2 주요 색상
- 스트라우만 브랜드 색상 활용
- 상태별 색상 구분 (활성/비활성, 완료/대기 등)

### 4.3 컴포넌트
- 버튼, 폼, 카드, 모달 등 재사용 가능한 컴포넌트 설계
- 일관된 spacing 및 typography

## 5. 비기능적 요구사항

### 5.1 성능
- 페이지 로딩 시간 2초 이내
- API 응답 시간 500ms 이내

### 5.2 보안
- 회원 전용 기능은 인증 필수
- 결제 정보는 PG사를 통해 안전 처리
- QR코드 정보는 암호화

### 5.3 호환성
- 최신 브라우저 (Chrome, Edge, Safari, Firefox)
- 모바일 브라우저 (iOS Safari, Chrome Mobile)
